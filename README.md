# office-ng28-archive

## 项目简介

`office-ng28-archive` 是一个用于归档与发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅用于存放和展示一系列独立的静态页面资源。

## 目录说明

```
.
├── index.html          # 仓库主页 / 导航页
├── pages/              # 存放独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要归档目录，每个 HTML 文件为一个独立页面。
- `assets/`：存放页面共用的样式、脚本和媒体资源。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可直接在浏览器中打开。
- 页面之间无依赖关系，可单独访问。
- 归档内容可能包含静态演示、文档、工具页面或信息展示页。
- 本仓库不保证页面内容的长期可用性或完整性，仅供归档参考。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 使用浏览器打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 也可通过 GitHub Pages 或其他静态托管服务部署后在线访问。

## 维护说明

- 本仓库由维护者不定期更新。
- 页面内容可能因归档需要而调整或删除。
- 欢迎提交 Issue 或 Pull Request 进行建议或贡献，但请注意：
  - 不涉及具体商业、营销或推广内容。
  - 不添加任何域名跳转或诱导点击元素。
  - 新增页面需保持独立、简洁、无外部依赖。
